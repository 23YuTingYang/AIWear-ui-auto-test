# 登录页面测试用例

访问地址：`http://139.199.222.234/login`

说明：本文档按当前 `UI自动化测试/AIWear_ui_auto_test/tests/test_login.py` 的实际校验逻辑整理。

测试前置说明：

- `baseUrl` 为 `http://139.199.222.234`
- 验证码登录依赖 Redis 读取验证码；若 Redis 不可用，当前脚本会直接跳过 `login_code_success`。
- 当前脚本未实现“验证码过期后登录失败”场景，该项已从文档中移除。
- 当前脚本在成功登录后通过页头检查“邮箱或用户名任一可见”，不是同时强校验两者。

执行顺序说明：

1. 当前脚本按固定顺序执行：默认验证码表单 -> 切换密码表单 -> 邮箱格式错误 -> 错验证码 -> 验证码成功 -> 刷新保持登录态 -> 错密码 -> 退出后访问受保护页 -> 密码成功登录。
2. 顺序来源于 `CASE_ORDER`，不是按文档优先级自动排序。

## P0

测试用例名字：1. 登录页面-验证码登录成功并跳转到图片编辑页  
操作步骤：

1. 打开 `http://139.199.222.234/login`
2. 保持“验证码登录”模式
3. 输入测试邮箱
4. 点击发送验证码
5. 从 Redis 读取验证码
6. 输入验证码并提交
   预期结果：
7. 页面跳转到 `/edit`
8. 顶部标题为“图片编辑”
9. 页头可见当前登录用户信息

测试用例名字：2. 登录页面-账号密码登录成功并跳转到图片编辑页  
操作步骤：

1. 打开登录页
2. 切换到“密码登录”
3. 输入正确用户名和密码
4. 点击提交
   预期结果：
5. 页面跳转到 `/edit`
6. 顶部标题为“图片编辑”
7. 页头可见当前登录用户信息

测试用例名字：3. 登录页面-正确邮箱加错误验证码登录失败  
操作步骤：

1. 打开登录页
2. 保持“验证码登录”模式
3. 输入正确邮箱
4. 输入错误验证码
5. 点击提交
   预期结果：
6. 页面仍停留在 `/login`
7. 页面出现错误提示
8. 页头不显示登录态

测试用例名字：4. 登录页面-正确账号加错误密码登录失败  
操作步骤：

1. 打开登录页
2. 切换到“密码登录”
3. 输入正确用户名和错误密码
4. 点击提交
   预期结果：
5. 页面仍停留在 `/login`
6. 页面出现错误提示
7. 页头不显示登录态

## P1

测试用例名字：5. 登录页面-邮箱格式错误时不能作为验证码发送前置条件  
操作步骤：

1. 打开登录页
2. 保持“验证码登录”模式
3. 输入错误邮箱格式
4. 点击发送验证码
   预期结果：
5. 页面仍停留在 `/login`
6. 页面不出现“验证码已发送”成功文案

测试用例名字：6. 登录页面-登录成功后刷新仍保持登录态  
操作步骤：

1. 基于已登录状态访问 `/edit`
2. 刷新页面
   预期结果：
3. 页面仍停留在 `/edit`
4. 顶部标题为“图片编辑”
5. 页头仍显示登录态

测试用例名字：7. 登录页面-退出登录后访问受保护页面跳回登录页  
操作步骤：

1. 使用正确用户名密码登录
2. 进入 `/edit`
3. 执行退出登录
4. 再访问 `/edit`
   预期结果：
5. 退出后页面回到 `/login`
6. 再访问受保护页面时仍被重定向到 `/login`
7. 页头不显示登录态
