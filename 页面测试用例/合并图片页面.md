# 合并图片页面测试用例

访问地址：`http://139.199.222.234/merge`

说明：本文档按当前 `UI自动化测试/AIWear_ui_auto_test/tests/test_merge.py` 与页面对象的实际行为整理。

测试前置说明：

- 访问前需先完成登录。
- 当前脚本通过图片选择弹窗选图，不是本地文件上传。
- 当前脚本会根据 `image1_key` / `image2_key` 读取配置中的图片路径，并取文件名主体传入页面对象进行弹窗选图。
- 当前用例中图片1使用 `person_female`，图片2使用 `cloth_red`，合并指令为 `给图1的人物换上图2的衣服`。

## P0

测试用例名字：1. 合并图片页面-上传两张图片并输入描述后可以提交  
操作步骤：

1. 登录后打开 `http://139.199.222.234/merge`
2. 按 `image1_key` 选择图片1
3. 按 `image2_key` 选择图片2
4. 输入描述 `给图1的人物换上图2的衣服`
5. 点击“提交”
   预期结果：
6. “提交”按钮可点击
7. 点击提交后页面出现 `AI计算中...` 加载文案

测试用例名字：2. 合并图片页面-提交成功后展示合并结果图  
操作步骤：

1. 登录后打开合并图片页面
2. 按 `image1_key` 选择图片1
3. 按 `image2_key` 选择图片2
4. 输入描述 `给图1的人物换上图2的衣服`
5. 点击“提交”
6. 等待合并结果图出现
   预期结果：
7. “提交”按钮可点击
8. 页面展示 `.merge-result-img` 合并结果图

测试用例名字：3. 合并图片页面-缺少图片1时提交按钮保持不可点击  
操作步骤：

1. 打开合并图片页面
2. 只按 `image2_key` 选择图片2
3. 输入有效描述
   预期结果：
4. “提交”按钮不可点击

测试用例名字：4. 合并图片页面-缺少图片2时提交按钮保持不可点击  
操作步骤：

1. 打开合并图片页面
2. 只按 `image1_key` 选择图片1
3. 输入有效描述
   预期结果：
4. “提交”按钮不可点击

## P1

测试用例名字：5. 合并图片页面-缺少描述时提交按钮保持不可点击  
操作步骤：

1. 打开合并图片页面
2. 按 `image1_key` 和 `image2_key` 分别选择图片1、图片2
3. 不输入描述
   预期结果：
4. “提交”按钮不可点击

4. 默认示意图 `.merge-default-img` 可见

测试用例名字：7. 合并图片页面-左侧菜单高亮显示图片合并  
操作步骤：

1. 打开合并图片页面
   预期结果：
2. 左侧当前高亮菜单文本为“图片合并”

测试用例名字：8. 合并图片页面-右上角用户下拉菜单可以展开  
操作步骤：

1. 打开合并图片页面
